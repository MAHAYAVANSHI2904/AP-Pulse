Architecture

How it works.

01

PDF / Image Upload

Streamlit file uploader accepts PDF, PNG, JPG. PyMuPDF converts PDF pages to high-resolution PNG (200 DPI). OpenCV deskews, denoises, and binarizes each page for optimal OCR accuracy.

02

3-Pass AI Extraction

Pass 1: Groq LLaMA vision extracts all fields. Pass 2: Math validation cross-checks CGST=SGST, grand total, line item sum. Pass 3: Vendor script generated for reuse on future invoices from the same vendor.

03

Compliance Engine

Python rules engine runs 30+ checks — no external compliance APIs. TDS section resolved via priority cascade: Invoice Nature → SAC code → AI prediction → manual review flag. 194C aggregate tracked across sessions via SQLite.

04

Audit Logging

Every decision (pass/fail/flag) written to SQLite and Google Sheets. Compliance score (0–100), reasoning, and action checklist stored per invoice per user session.

05

Dashboard + Export

Tabbed Streamlit UI: Overview, Forensic Scans, TDS Summary, GST Summary, Vendor Data, Journal Entries. Export: CSV, Tally XML. GSTR-2B upload for ITC reconciliation.

Setup

Running locally.

Step 1
Clone the repo
terminal bash 
$ git clone https://github.com/MAHAYAVANSHI2904/AP-Pulse.git
$ cd AP-Pulse
$ pip install -r requirements.txt
Step 2
Configure secrets
Create .streamlit/secrets.toml — never commit this file.
.streamlit/secrets.toml toml 
GROQ_API_KEY   = "gsk_your_key_here"
SHEET_ID       = "your_google_sheet_id"
SELF_GSTIN     = "your_company_gstin"
SELF_PAN       = "your_company_pan"

[gcp_service_account]
type           = "service_account"
project_id     = "your_project_id"
private_key    = "-----BEGIN RSA PRIVATE KEY-----\n..."
client_email   = "sa@project.iam.gserviceaccount.com"
Step 3
Run the app
terminal bash 
$ streamlit run Invoice_Hub.py
# → http://localhost:8501

Security

No secrets in source.

🔑

API Keys via st.secrets

Groq API key and Google SA credentials managed via Streamlit secrets or env vars only. Never committed to repo.

🗂️

DB Files Excluded

invoice_forensics.db and vendor_intelligence.json are gitignored. No invoice data in repo.

🏢

Org Identifiers via Env

Company GSTIN and PAN loaded from environment variables, not hardcoded in source code.

🧹

No Real Vendor Data

All sample invoices in this repo use dummy placeholder values. No real financial records stored.
