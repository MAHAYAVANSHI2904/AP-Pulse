The Problem

Why AP Pulse was built

Finance teams at NBFCs and CA firms were losing 2+ hours every day to manual invoice entry, error-prone GST and TDS calculations, and fragmented ledger updates. Every invoice required manual field extraction, manual tax validation, and manual data entry into financial records — creating a high-error, high-effort process that scaled poorly with volume.

AP Pulse was built after directly observing these pain points inside a live NBFC environment, identifying invoice processing as the single highest-friction function in the AP cycle.

How It Works — Architecture

📄

Step 1 — Upload

User uploads PDF or scanned image invoice via Streamlit UI

🔍

Step 2 — OCR Extraction

PyMuPDF + Tesseract extracts raw text from invoice with AI self-correction fallback

🧠

Step 3 — LLM Parsing

Groq AI / Gemini Flash / Claude API parses structured fields from extracted text

✅

Step 4 — Compliance Check

Validates GSTIN, PAN, MSME/Udyam, auto-calculates TDS, checks GST summary

📊

Step 5 — Ledger Sync

Pushes verified vendor data, tax figures and journal entries to Google Sheets via API

📁

Step 6 — Export

Generates Tally XML export and GSTR-2B reconciliation output

Key Features

  • End-to-end OCR + LLM pipeline — parses complex PDF and image invoices with AI self-correction fallback (Groq, Gemini Flash)
  • Smart compliance engine — validates GSTIN, PAN, MSME/Udyam registration and auto-calculates TDS producing audit-ready output
  • Google Sheets API integration — pushes verified vendor data, tax figures and journal entries directly, eliminating manual re-entry
  • GSTR-2B auto-reconciliation — matches purchase data against GSTR-2B for ITC claim accuracy
  • Tally XML export — direct export format compatible with Tally Prime import
  • Multi-user SQLite architecture — isolated login with role-based access control
  • AI fallback layer — when OCR confidence is low, LLM self-corrects extraction errors automatically

Security

Security Measures Applied

  • No API keys, credentials or secrets committed to this repository
  • All secrets managed via st.secrets (Streamlit Cloud) or environment variables
  • .gitignore configured to exclude .env files, database files, and credential files
  • No real invoice data, vendor names or financial records stored in this repo
  • SheetsConnector.py uses environment-based Sheet ID — no hardcoded values
  • All sample/test data uses dummy placeholder values only
